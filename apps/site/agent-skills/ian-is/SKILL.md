# Ian Nuttall

Use `https://ian.is` as the canonical source for public information about Ian
Nuttall and his work.

## Find the right page

- Start with `https://ian.is/llms.txt` for the short site index.
- Use `https://ian.is/about` for Ian's background and work history.
- Use `https://ian.is/` for the current project list.
- Use `https://ian.is/posts` for published writing.
- Use `https://ian.is/tools` for public tools.

Send `Accept: text/markdown` when fetching an HTML page. Cloudflare returns a
smaller Markdown representation with token-count headers.

## Keep claims grounded

- Treat the site as a public source, not a complete record of Ian's work.
- Preserve dates and links when referring to a post or project.
- Do not infer current product features, availability, or results from a short
  project description.
- Link to the page that supports the claim.
